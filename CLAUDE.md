# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

A static site + downloadable kit (`brand-in-a-box.zip`) that turns a 10-question intake into a complete brand design system inside Claude. Built for the Purely Personal workshop. Auto-deployed to https://claude-design-beginners.vercel.app via Vercel on push to `main`.

There is no build step. No tests. No lint. The site is hand-written HTML/CSS/JS and the kit is markdown + HTML. The deliberate simplicity is load-bearing — see "Things that are intentional" below.

## Common commands

```bash
# Rebuild brand-in-a-box.zip after editing intake/ or examples/

# Local preview of the form (no server needed — file:// works)

```

## Architecture: two outputs from shared sources

The repo produces two distinct artifacts that draw from the same source files:

1. **`public/brand-in-a-box.zip`** — what attendees download. Generated by `scripts/bundle-zip.sh` from `intake/` + `examples/`. Never edit the zip directly — always edit the source folders and rebundle.

2. **`public/index.html`** — what attendees use online. Single-file HTML with localStorage autosave, a 9-family live brand preview that updates colors/fonts/canvas as you type, and a "Generate Claude prompt" button that composes their answers into a paste-ready prompt.

The same form ships in both places: `public/index.html` (live web) and `intake/intake-form.html` (offline copy bundled in the zip). The bundle script does NOT auto-sync these — you must `cp public/index.html intake/intake-form.html` if you change the form.

## How the brand generation actually works

The flow attendees follow:

1. Fill `INTAKE.md` (10 questions) or use `intake-form.html` (same, with live preview)
2. Paste their answers into Claude with the kit folder loaded
3. The agent reads `intake/SKILL.md` and generates a complete `{business-slug}/` folder

`intake/SKILL.md` is the load-bearing document. It defines:

- An **inference table** mapping 12 missing fields (anti-positioning, voice rules, font defaults, banned phrases, etc.) to the answers they're derived from.
- **Voice DNA extraction** from the user's three writing samples (Q7) — the SKILL never asks voice questions, it derives them empirically. This is intentional and load-bearing — the lean intake works because samples reveal voice better than self-reported preferences. Do not add voice questions back to the intake.
- **Advisor mode** — when the family choice and accent disagree (e.g. hot red + Cult/Indie), the SKILL stops and proposes 3 directions instead of guessing.
- **Anti-slop rules** — universal hard bans (no SaaS gradients, no glassmorphism, no em dashes, no emoji bullets, etc.).
- **Optional deepening pass** — three opt-in lanes (deepen positioning / icp / voice) that let founders add depth post-generation without bloating the intake.

## Three example brands serve as proof, not templates

- `examples/purely-personal/` — Editorial Minimal (sans, hot red `#e90d41`, 0px corners, 9-word sentences). The founder's real brand.
- `examples/marigold-and-oak/` — Warm Editorial (Crimson Pro serif, terracotta `#c8543b`, 8px corners, 18-word flowing sentences). Fictional wellness consultancy.
- `examples/voltage-labs/` — Cinematic Dark (Space Grotesk, electric cyan `#00d4ff`, void canvas, 11-word declarative sentences, mono eyebrows). Fictional B2B AI agency.

Each example folder mirrors what the SKILL would generate for that brand: `brand/` (positioning, voice-dna, icp, colors, typography, tokens.json, anti-slop), `colors_and_type.css`, personalized `SKILL.md`, `DESIGN.md`, `README.md`, and one HTML sample. **Same SKILL produced all three.** Don't add a fourth without good reason — three demonstrates range cleanly.

When creating new examples, do not template — write each brand's voice and tokens with intention. The point of these is that they read as real brands, not parameterized mocks.

## Live preview architecture

`public/index.html` contains a `FAMILY_PRESETS` JS object with one entry per aesthetic family. Each preset has `canvasBg`, `text`, `textMuted`, `accent`, `headingFont`, `bodyFont`, `pillRadius`. The `updatePreview()` function applies the preset to the preview panel on every form change, plus an accent override if the user typed a valid hex.

If you add a new family to the dropdown in `<select id="family">`, you must also add a matching entry to `FAMILY_PRESETS`. Otherwise the preview falls back to Editorial Minimal.

All 9 families' fonts are loaded eagerly via a single Google Fonts URL in the `<link>` tag. This is a deliberate trade — page load is heavier, but family switching in the preview is instant.

## Vercel config notes

`vercel.json` sets `outputDirectory: public`, `cleanUrls: true`, and redirects `/design-intake` and `/design-intake.html` to `/` (back-compat for early shared links). OG meta tags in `public/index.html` reference `/og-image.png` at the production URL — if the URL changes, update both `vercel.json` and the meta tags.

## Things that are intentional, not bugs

- **No build step, no framework, no Node runtime in production.** The kit must work offline when attendees unzip it. Adding a build defeats the point.
- **The HTML form is one ~700-line file.** Resist splitting into components. The form ships inside the zip and runs from `file://`.
- **`scripts/bundle-zip.sh` is plain bash.** No `npm run build`, no fancy tooling. Same offline-portability reason.
- **Three example brands ship as separate folders, not a templating system.** Showing rendered output IS the demo. Generating examples on-demand would undercut the message.
- **Voice DNA in `examples/*/brand/voice-dna.md` references "extracted from your samples".** That's the SKILL behavior we want — empirical voice rules with cited sources, not vibes. Preserve that pattern when editing examples.
- **`docs/demo.gif` and `public/og-image.png` are committed binaries.** They're deploy artifacts referenced by README and meta tags. Regenerate via the commands above when the source HTML changes.

## Known issues

- `scripts/record-demo.js` errors at the `#pain` field click because section 2 of the form is collapsed by default. Fix: before any `typeInto` call, force all `.section` elements to have the `open` class via `page.evaluate`. Once fixed, output goes to `.demo-frames/` and needs ffmpeg to stitch into `docs/demo.gif`.

## When editing prose in this repo

The README and the example `brand/voice-dna.md` files are written in the Purely Personal voice (one sentence per paragraph, no em dashes, "I" + "you", outcome-first with numbers, sentence case, no banned phrases like `growth hack`, `10x`, `leverage`, `unlock`, `secret sauce`). Keep that voice when adding to user-facing copy. The example brands have their own distinct voice rules in their respective `voice-dna.md` files — match the brand you're editing.
