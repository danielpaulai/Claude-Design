## How to use this folder

Three lanes.

**Lane 1: Inside Claude Code or Claude Desktop.**
Drop this folder into a session. The SKILL activates automatically. Ask for posts, slides, or copy. Output stays in voice and on-brand.

**Lane 2: Inside Claude Design.**
Open `DESIGN.md`. Upload to claude.ai/design under "Add assets." Every project from now on inherits this design.

**Lane 3: As source for any other tool.**
`brand/tokens.json` is the source of truth. Regenerate Figma tokens, Canva brand kits, or CSS from it.

## What is in `brand/`

- `positioning.md` — one-line positioning, category, anti-positioning, alternatives table, enemies.
- `voice-dna.md` — voice rules extracted from your three samples in Q7. Pronouns, rhythm, banned phrases, hook mechanisms.
- `icp.md` — your ideal client profile, expanded from Q3 + Q4 + Q5.
- `colors.md` — narrative version of brand colors with usage rules.
- `typography.md` — narrative version of the type system with hard rules.
- `tokens.json` — design tokens (DTCG format). Source of truth for all downstream tools.
- `anti-slop.md` — what NOT to do. Universal anti-slop plus brand-specific bans.

## What this folder is and is not

This is a starter.

It is not a finished brand.

The next 7 days, revisit:

- `voice-dna.md` after you have written 10 more posts. Does the rhythm rule still hold?
- `anti-slop.md`. Add brand-specific imagery rules as you ship more artifacts.
- Logo. The system has no SVG logo yet. Commission one or generate a typographic mark.
- `samples/`. Fill in any format you did not tick in Q8 once you need it.

You will know the system is working when you stop reaching for Canva.
